# 简介

每一道菜都是独特的作品，不断尝试，利用各种食材和烹饪手法重现经典，造就新锐。

玩家可以向“锅具”内加入各种各样的食材，兼容其他mod。

# 味觉系统

## 食物属性

一个食物（不是原材料）的nbt属性：

- 食材[list]
  - string:物品id
  - int:数量单位，默认为1
  - 食材处理顺序（影响回复的饱食度和饱和度）[list]
    - int:烹饪手法（详见玩法-烹饪）
  - 气味[list]
    - string:气味名称（每种气味都会发出粒子）
    - int:气味的强度
  - 口味[list]
    - string:口味名称（）
    - int:口味的强度

在mod的config文件夹中，包含对原版&模组食材的属性定义，在制作饭菜时使用到这些值：

*文件名=物品id*
- 气味
- 口味

## 嗅觉&味觉

嗅觉与味觉实际上由大量的香料&调料组成，其中，嗅觉影响食用后产生的粒子效果：
1. 每种嗅觉提供一种新的粒子效果
2. 粒子颜色由食物的所有气味代表的颜色平均值决定
3. 粒子数量由气味的强度平均值得出
4. 粒子持续时间由所有气味的强度之和决定

香料&调料可以在腌制/混合/炒制时加入。
可能会添加几十种香料&调料。

## 特殊食材：汤汁

如果加入了水，会根据加入的量增加汤汁的量。
汤汁不是物品，如果nbt中有汤汁材料，则混合所有汤汁，并且在容器内渲染。
每个单位的汤汁会增加1s的食用速度。

# 玩法

## 烹饪

影响回复的饱食度与饱和度以及食用速度，
但在这之前，需要对食材进行一些小小的处理：

1. 切块（食用速度增加20%，熟成速度增加20%）
2. 切片-加40，30
3. 切丝-加55，40
4. 切丁-加65，50
5. 腌制-每种材料减少50%饱和度损耗，非线性叠加
6. 混合-添加其他食材

经过以下任一烹饪手法，模组会尝试查找食物的熔炉配方，并合成（比如铁矿放锅里炸一炸就成了铁锭）

1. 蒸（锁住营养，饱和度+1）
2. 煮（锁住营养，饱食度+1）
3. 炸（食用速度增加）
4. 炒（允许在炒的时候加入其他食材，包括水，新食材会继承之前食材的嗅觉）
5. 烤（下方有光源即可，比如火把）

## 食用

所有该模组内的食物都会在食用时读取nbt内容，并进行结算：

0. 食用速度（叠加，每个单位量的食材线性增加食用时间）
1. 回复的饱食度（叠加）
2. 回复的饱和度（叠加）
3. 从原食材那继承的buff（切割后buff时间会相应变化）
4. 粒子特效

### 饱腹

饱食度已满的情况下可以继续吃，若饱和度也是满的，就增加伤害吸收
